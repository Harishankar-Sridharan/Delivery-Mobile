# mobile_loading
# mobile

The Nomadia Delivery mobile loading feature allows delivery users to manage parcel statuses and track loading activities. Users can verify parcels and machines before starting their routes to ensure efficient preparation. This workflow streamlines the transition from the warehouse to the vehicle.

### Getting Started

*   ND mobile application installed on the device.
*   An assigned route available in the application.

1.  Open the assigned route from the **Dashboard**.

![Flow 0:13 to 0:19](../images/mobile-mobile_loading_timestamp_0_to_13_to_0_to_19.gif "Dashboard – The user selects an assigned route to begin.")

2.  Select **Load my vehicle** to start the process.

![Frame at 0:25](../images/mobile-mobile_loading_timestamp_0_to_25.png "Load my vehicle – The button initiates the loading interface.")

### Feature Overview

*   **To Load**: Displays all parcels and machines pending loading.

![Frame at 0:33](../images/mobile-mobile_loading_timestamp_0_to_33.png "To Load section – A list showing items waiting to be loaded.")

*   **Not Loaded**: Lists parcels that are not yet inside the vehicle.

![Frame at 0:41](../images/mobile-mobile_loading_timestamp_0_to_41.png "Not Loaded section – A specific view for items not yet scanned.")

*   **Loaded**: Shows parcels that are successfully loaded into the vehicle.

![Frame at 0:49](../images/mobile-mobile_loading_timestamp_0_to_49.png "Loaded section – A list of items confirmed as loaded.")

*   **Sub-status Toggle**: Switches the view to verify loaded and pending parcels.

![Frame at 0:57](../images/mobile-mobile_loading_timestamp_0_to_57.png "Sub-status Toggle – The button used to switch between parcel statuses.")

*   **Ellipse Menu**: Opens additional operation options in the top right corner.

![Frame at 1:36](../images/mobile-mobile_loading_timestamp_1_to_36.png "Ellipse Menu – The three-dot icon containing advanced loading settings.")

### How To: Load Parcels via Scanning

1.  Access the scan functionality to load parcels quickly.

![Frame at 1:06](../images/mobile-mobile_loading_timestamp_1_to_06.png "Scan button – The icon used to activate the barcode scanner.")

2.  Scan the parcel barcode to automatically update the loading status.

![Flow 1:06 to 1:11](../images/mobile-mobile_loading_timestamp_1_to_06_to_1_to_11.gif "Barcode scanning – The camera scanning a parcel to update its status.")

### How To: Load Parcels Manually

1.  Identify the parcel entry in the list if a physical scan is unavailable.
2.  Long press on the parcel entry to update the status manually.

![Flow 1:19 to 1:24](../images/mobile-mobile_loading_timestamp_1_to_19_to_1_to_24.gif "Manual Loading – A long press on an item to mark it as loaded.")

### How To: Finish the Loading Workflow

1.  Verify all loaded parcels and machines in the list.
2.  Select **Validate** to confirm loading completion.

![Frame at 2:26](../images/mobile-mobile_loading_timestamp_2_to_26.png "Validate button – The button to finalize the loading session.")

3.  Tap on **Confirm** when the pop-up asks to finish loading.

![Frame at 2:36](../images/mobile-mobile_loading_timestamp_2_to_36.png "Confirmation pop-up – The final prompt to close the loading workflow.")

#### Troubleshooting: Unexpected Machines
If an unexpected parcel or machine is scanned, the application identifies it as an **Unexpected machine**.

![Frame at 2:18](../images/mobile-mobile_loading_timestamp_2_to_11_to_2_to_18.png "Unexpected machine alert – A notification for items not in the route.")

### Productivity Tips

- 💡 **Reset Progress**: Use **Reset all packages** in the ellipse menu to clear all statuses and restart.
- 💡 **Cleaner View**: Use the **Configuration** option to minimize operation panels for a cleaner interface.
- ⚠️ **Scanning Issues**: Perform a manual scan via long press if the physical barcode scanner fails.
